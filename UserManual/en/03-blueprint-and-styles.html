3. WidgetBlueprint, Variables and Style Sheets

Configure the generated blueprint — pick a parent class, implement interfaces, declare blueprint variables, and attach reusable style sheets.

3.1 The <WidgetBlueprint> root

A markup file usually starts with a WidgetBlueprint element. It accepts the special Super attribute to choose a parent class, and optional Implements / StyleSheets attributes: 

<WidgetBlueprint
    Super="UserWidget"
    Implements="MyInterface,MyOtherInterface"
    StyleSheets="/Game/UI/Themes/DarkTheme">
  <WidgetTree>
    <CanvasPanel />
  </WidgetTree>
</WidgetBlueprint>

Super

Resolves to a Unreal class via the type parser. The default is UserWidget. You can pass any reachable class name; for custom blueprint base classes, use the short class name.

Implements

Comma-separated list of interface names. Each name is resolved as a class derived from UInterface; on hot reload the existing implementation list is cleared and rebuilt to match the XML, so removing an interface from the file removes it from the compiled class.

StyleSheets

Comma- or semicolon-separated list of UWidgetStyleSheet asset paths. The plugin copies the asset content into an inline structure on the blueprint at compile time, so the runtime does not keep a hard reference to the asset. See § 3.4 for details.

3.2 Declaring variables

Inside <WidgetBlueprint> you can place <Variable> elements to add member variables on the generated class: 

<WidgetBlueprint>
  <Variable Name="Health"   Type="Float"  DefaultValue="100" />
  <Variable Name="Tags"     Type="Array<Name>" />
  <Variable Name="Selected" Type="MyDataAsset" />
</WidgetBlueprint>

The Type string is parsed by FTypeParser:

  • Built-in scalars: Bool, Byte, Int32, Int64, Float, Double, String, Text, Name.
  • Structs: any USTRUCT, e.g. Vector2D, LinearColor.
  • Enums: any UENUM.
  • Object references: any UCLASS short name.
  • Containers: Array<T>, Set<T>, Map<K,V>. Container nesting is not supported; Array<Array<Int>> is rejected.

3.3 Naming widgets for blueprint access

As shown in chapter 2, the Name attribute on a widget promotes it to a blueprint variable. Combine that with <Variable> declarations to expose a clean public API to the rest of your project: 

<WidgetBlueprint Super="UserWidget">
  <Variable Name="Title" Type="Text" />
  <WidgetTree>
    <VerticalBox>
      <TextBlock Name="TitleLabel" Text="Placeholder" />
      <Button    Name="PrimaryButton" />
    </VerticalBox>
  </WidgetTree>
</WidgetBlueprint>

3.4 Style sheets

A UWidgetStyleSheet asset is a list of (WidgetName, PropertyPath, Value) entries. When you apply one to a UserWidget, each entry locates the widget by name in the widget tree and writes the value through the same property pipeline used at compile time.

Reference style sheets in two ways:

  1. Top-level StyleSheets="..." attribute on the <WidgetBlueprint>: copied into the generated class extension and applied automatically when the UserWidget is initialized.
  2. Programmatically through UWidgetMarkupUserWidgetExtension::SetStyleSheets() or UWidgetStyleSheet::ApplyToUserWidget() if you want to swap themes at runtime.
Failures while applying an individual style do not stop the rest of the sheet from being applied; the editor logs a warning per failed entry.
