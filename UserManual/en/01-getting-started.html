1. Getting Started

Enable the plugin, create your first *.widgetmarkup file, and open the live preview window.

1.1 Requirements

  • Unreal Engine 5.x with the editor available.
  • The WidgetMarkup plugin enabled in your project.
  • (Optional) The WidgetMarkupPythonIntegration plugin, which itself depends on the built-in PythonScriptPlugin.
  • (Optional) The VS Code extension for syntax checking and IntelliSense.

1.2 Enable the plugins

  1. Open your project in the Unreal Editor.
  2. Go to Edit → Plugins.
  3. Find WidgetMarkup and enable it. Restart the editor when prompted.
  4. If you plan to use Python view-models, also enable Python Editor Script Plugin and WidgetMarkupPythonIntegration, then restart again.

1.3 Create your first markup file

Create a plain text file in your project's content tree, with the extension .widgetmarkup. For example: 

Game/Content/WidgetMarkup/Example.widgetmarkup

Paste the following minimal example:

<WidgetBlueprint>
  <WidgetTree>
    <CanvasPanel>
      <TextBlock
          Text="Text Block in Canvas Panel"
          Slot.LayoutData.Anchors.Minimum="0.5,0.5"
          Slot.LayoutData.Anchors.Maximum="0.5,0.5"
          Slot.bAutoSize="True"
          Slot.LayoutData.Alignment="0.5,0.5" />
    </CanvasPanel>
  </WidgetTree>
</WidgetBlueprint>

1.4 Open the preview window

In the Unreal Editor, focus the CMD input bar at the bottom and run: 

WidgetMarkup.Show /Game/WidgetMarkup/Example

The argument is a package path in Unreal asset format, starting with /Game/... and without the .widgetmarkup extension. The plugin resolves it to the source file on disk, compiles it, and opens SWidgetMarkupWindow with a live preview.

Once the window is open the file is being watched: every time you save changes from your editor, the preview recompiles and refreshes automatically.

1.5 Generate IntelliSense data (optional)

For autocomplete and basic schema validation in VS Code, export the type metadata once per project: 

WidgetMarkup.GenerateIntellisenseData <output-folder>

Then point the IntelliSense Data Path setting of the VS Code extension to that folder.

1.6 What you just built

Behind the scenes, the editor created a transient UWidgetBlueprint, walked your XML tree, instantiated a UCanvasPanel with a UTextBlock child, applied each property, and finally compiled the blueprint. The preview window simply hosts the resulting widget. The next chapter explains how the XML maps to UMG types and properties.
