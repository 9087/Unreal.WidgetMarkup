4. Python Integration Overview

Attach a Python class as the view-model behind any markup-built UserWidget, with one line of XML.

4.1 What the integration plugin does

WidgetMarkupPythonIntegration is an editor module that depends on both WidgetMarkup and the built-in PythonScriptPlugin. It listens for the WidgetMarkup user widget initialized event raised when a UserWidget compiled from a markup file is instantiated, and for any widget whose markup declares a non-empty Script attribute it:

  1. imports the Python module named by Script;
  2. resolves the class with the same short name as the module;
  3. instantiates that class via WidgetMarkupComponent.create();
  4. stores the Python instance on a per-widget extension so its lifetime tracks the UserWidget.

4.2 Enable the plugin

  1. Make sure the editor was built with Python support (WITH_PYTHON defined). Standard editor builds have it.
  2. In Edit → Plugins, enable Python Editor Script Plugin.
  3. Enable WidgetMarkupPythonIntegration and restart the editor.
If Python is unavailable at runtime the plugin logs a warning and skips component creation. The widget itself still works; only the Python view-model is missing.

4.3 Where Python modules live

The integration plugin ships its base classes under: 

Game/Plugins/WidgetMarkupPythonIntegration/Content/Python/
    WidgetMarkupComponent.py
    ObservableCollection.py

The standard PythonScriptPlugin already adds these folders to sys.path at editor startup. Place your own component modules either in the same folder, in a project-level Content/Python directory, or anywhere reachable on sys.path through your init_unreal.py.

4.4 The Script attribute

Reference your Python class from XML with one attribute on the <WidgetBlueprint> root: 

<WidgetBlueprint Script="my_game.ui.HealthBar">
  <WidgetTree>
    <HorizontalBox>
      <ProgressBar Name="HealthFill" />
      <TextBlock   Name="HealthText" />
    </HorizontalBox>
  </WidgetTree>
</WidgetBlueprint>

The string is treated as a fully qualified module path. The component class name is derived from the last segment, so the example above would import my_game.ui.HealthBar and instantiate the class HealthBar defined in it.

4.5 A minimal Python component

Create my_game/ui/HealthBar.py on the Python path: 

from WidgetMarkupComponent import WidgetMarkupComponent, reactive, computed


class HealthBar(WidgetMarkupComponent):
    @reactive
    def current(self):
        return 100

    @reactive
    def maximum(self):
        return 100

    @computed
    def percent(self):
        return self.current / max(self.maximum, 1)

    def on_property_changed(self, property_name, value):
        # Push values to the bound widget here.
        # See chapter 5 for full reactive semantics.
        print(f"{property_name} -> {value}")

Once the markup compiles and an instance of the UserWidget is created (for example by adding it to the viewport), the integration constructs a HealthBar, owns it for the lifetime of the widget, and starts dispatching change notifications.

4.6 Lifecycle summary

StageWhat happens
Editor startupBoth plugins load; the integration subscribes to the WidgetMarkup initialization event.
Markup compileThe Script value is stored on a blueprint extension and copied into the generated class.
UserWidget initializeThe integration imports the module, calls WidgetMarkupComponent.create(script) and pins the Python instance to a per-widget extension.
UserWidget destroyThe extension is released and the Python reference count drops; the GIL is acquired before Py_DECREF.

4.7 Running the included tests

The plugin ships unit tests for the reactive base class. From the editor's Python REPL you can run, for example: 

import unittest, sys, os
sys.path.insert(0, os.path.join(unreal.Paths.project_plugins_dir(),
    "WidgetMarkupPythonIntegration/Content/Python/tests"))
loader = unittest.TestLoader()
suite  = loader.discover(start_dir=sys.path[0], pattern="test_*.py")
unittest.TextTestRunner(verbosity=2).run(suite)
