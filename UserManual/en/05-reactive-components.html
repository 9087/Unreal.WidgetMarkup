5. Reactive Python Components

Use @reactive, @computed and ObservableCollection to build small view-models that notify on every change.

5.1 The base class

WidgetMarkupComponent is the only base class your component needs to extend. It provides:

  • create(module_name) — the factory the C++ side calls.
  • prime_computed_dependencies() — eagerly evaluates every @computed property so its dependency graph is ready.
  • on_property_changed(name, value) — override this to forward changes to the bound widget.

5.2 @reactive properties

Decorate a method whose return value is the per-instance default. The decorator turns it into a settable descriptor that emits a change notification whenever the new value differs from the old: 

from WidgetMarkupComponent import WidgetMarkupComponent, reactive


class Counter(WidgetMarkupComponent):
    @reactive
    def value(self):
        return 0

c = Counter()
c.value = 1   # triggers on_property_changed('value', 1)
c.value = 1   # no notification: equal to the previous value

5.3 @computed properties

Decorate a method that derives its result from other reactive (or computed) properties. Dependencies are collected at runtime on every evaluation, so conditional branches are handled correctly: 

from WidgetMarkupComponent import WidgetMarkupComponent, reactive, computed


class Picker(WidgetMarkupComponent):
    @reactive
    def left(self):  return 0
    @reactive
    def right(self): return 0
    @reactive
    def use_left(self): return True

    @computed
    def selected(self):
        return self.left if self.use_left else self.right

After use_left flips to False, only changes to right will recompute selected; changes to left are ignored until use_left flips back. Computed properties are read-only.

If a computed property is never read before its dependencies change, it cannot announce its update because no dependency graph exists yet. Call self.prime_computed_dependencies() at the end of __init__ to build the graph eagerly.

5.4 Cycles and re-entrancy

  • Cyclic computed dependencies raise RuntimeError at evaluation time. The error message contains the offending path, for example first -> second -> first.
  • Inside on_property_changed you may write to other reactive properties; the resulting notifications are flushed before the original write returns.
  • If a computed function raises while recomputing, the cached value stays untouched and the next read attempts to recompute again.

5.5 Observable lists

Any plain Python list assigned to a @reactive property is automatically wrapped in an ObservableCollection. In-place mutations (append, extend, insert, pop, remove, clear, sort, reverse, slice assignment, and +=/*=) trigger a change notification: 

from WidgetMarkupComponent import WidgetMarkupComponent, reactive, computed


class Inventory(WidgetMarkupComponent):
    @reactive
    def items(self):
        return []

    @computed
    def item_count(self):
        return len(self.items)

inv = Inventory()
inv.prime_computed_dependencies()
inv.items.append('Sword')   # notifies 'items' and 'item_count'

5.6 Bridging values back to UMG

The integration plugin attaches your Python instance to the UserWidget but does not wire properties to widgets automatically. That keeps the binding policy in your hands: most projects override on_property_changed and call back into Unreal through unreal.find_object() / unreal.MyUserWidget.set_property(): 

import unreal


class HealthBar(WidgetMarkupComponent):
    def __init__(self):
        self.user_widget = None  # set by the host
        self.prime_computed_dependencies()

    @reactive
    def current(self): return 100
    @reactive
    def maximum(self): return 100

    @computed
    def percent(self):
        return self.current / max(self.maximum, 1)

    def on_property_changed(self, property_name, value):
        if not self.user_widget:
            return
        if property_name == 'percent':
            fill = self.user_widget.get_widget_from_name('HealthFill')
            if fill:
                fill.set_percent(value)
        elif property_name == 'current':
            text = self.user_widget.get_widget_from_name('HealthText')
            if text:
                text.set_text(unreal.Text(str(int(value))))
Future revisions of the integration plugin may add a built-in binding layer; for now, this hand-written pattern is the supported way to push reactive values into the widget tree.
