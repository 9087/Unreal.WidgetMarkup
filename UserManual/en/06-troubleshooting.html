6. Troubleshooting and Reference

Editor commands, common error messages, and pointers to the source code if you need to dig deeper.

6.1 Editor commands

CommandDescription
WidgetMarkup.Show <PackagePath> Compile the source file at the given /Game/... path and open the preview window. The file is then watched for changes.
WidgetMarkup.Compile <Name> <XML> Low-level helper that compiles XML directly from a string. Mostly useful for scripted tests.
WidgetMarkup.GenerateIntellisenseData <OutputPath> Export UMG type metadata as JSON for use by the VS Code extension.

6.2 Common errors

«Failed to resolve type»

Reported by FTypeParser when a class, struct or enum name in a Variable Type="...", Super or Implements attribute cannot be found. Use the short class name (no U/F prefix) and verify the asset/module is loaded.

«Container nesting is not supported: 'Array<Array<Int>>'»

Container types such as Array, Set and Map can only wrap a non-container element type. Replace the inner container with a struct or a custom type.

«Script property target is not a WidgetBlueprint»

The Script attribute is registered for UWidgetBlueprint. It only makes sense on the root <WidgetBlueprint> element.

«Failed to import component base module 'WidgetMarkupComponent'»

Either the WidgetMarkupPythonIntegration plugin is disabled, or its Content/Python directory is not on sys.path. Re-enable the plugin and restart the editor.

«Cyclic computed dependency detected»

A @computed property reaches itself through its dependencies. Refactor so that the cycle is broken — usually by promoting one node to a @reactive property and assigning to it explicitly.

6.3 Logs

  • LogWidgetMarkup — markup compilation, property resolution, style sheet application.
  • LogWidgetMarkupPythonIntegration — Python availability, module import errors, lifecycle.

Both categories default to Log; bump them to Verbose in the editor's Output Log filter to see lifecycle traces.
