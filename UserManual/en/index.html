WidgetMarkup User Manual

Build UMG widget trees in Unreal Engine with a small, declarative XML dialect, then drive them at runtime with reactive Python components.

This manual covers the WidgetMarkup editor plugin and its companion WidgetMarkupPythonIntegration plugin. It assumes you already know the basics of the Unreal Editor and UMG.

What is WidgetMarkup?

WidgetMarkup is an editor-only plugin that turns a *.unrealwidgetmarkup XML file into a compiled UMG widget tree (a UWidgetBlueprint in memory) and previews it live as you edit. You describe the widget hierarchy and the property values in XML; the plugin builds the matching Slate / UMG objects, watches the source file, and refreshes a preview window on every save.

WidgetMarkupPythonIntegration attaches a Python component instance to each UUserWidget created from a markup that declares a Script attribute. The Python class derived from WidgetMarkupComponent provides reactive properties, computed properties and observable lists, so you can keep your view-model in plain Python.

Conventions used in this manual. XML element and attribute names are written like <TextBlock Text="..." />. Editor commands are entered in the bottom-bar CMD box of the Unreal Editor.
